Getting Started With Hmix

It's pretty simple to get started with Hmix, but you'll want to install a few things to make development easier.

Regtest Mode

You'll need to have a Htmlcoin node running in order to deploy contracts. When developing contracts, we usually recommend running a node in regtest mode, a private development mode that allows you to quickly test contracts and transactions before deploying them to the Htmlcoin testnet or mainnet.

Building Contracts

Htmlcoin uses the Ethereum Virtual Machine, so smart contracts are written in Solidity. The best way to learn to write smart contracts is to read the documentation.

Compiling/Debugging

Contracts automatically compile as you write and save them, so you won't have to worry about manually compiling things. If you want to turn this off, head over to the compilation tab.

The compilation tab will show you any errors or warnings that the Solidity compiler spits out. You can change the current compiler version by selecting a new version from the dropdown in the compilation tab.

Deploying Contracts

Once you've written your contract, you'll probably want to deploy it and test the contract methods out. To do this, head over to the deploy/run tab. Here, you'll see an input field called "Htmlcoin RPC URL" and a dropdown that contains the contracts of whatever file you currently have open.

The Htmlcoin RPC URL input should point to the RPC URL of your node. If you used the above docker command to launch your Htmlcoin regtest node, then that URL should match the default http://htmlcoin:test@localhost:9888.

To deploy a contract, you'll first need to supply any constructor arguments you specified. These arguments are entered as a comma separated string. Then, you can hit the "deploy" button to deploy your contract to your Htmlcoin node.

Now, you should see a terminal log telling you that your contract has been deployed. You should also see your contract pop up in the run tab. This means you can start to send transactions and make contract calls.

Sending Transactions

Hmix allows you to test your contracts by sending transactions and making contract calls. All you need to do is enter arguments into the method's input field and hit the method's respective button.

Hmix automatically generates new blocks after each transaction so you'll instantly get a response back.
